# План реализации: Редизайн UX, инженерной логики и промышленная профессионализация HADALBORE_LAB

Этот документ содержит технический план модернизации приложения HADALBORE_LAB для перевода его из прототипа в профессиональный рабочий инструмент полевого инженера.

---

## User Review Required

> [!IMPORTANT]
> **Продукт и авторство**: Название продукта `HADALBORE_LAB` и создатель `Niko Y` остаются полностью неизменными во всех файлах конфигурации, предупреждениях и копирайтах. Все оффлайн-функции и защиты Field Mode будут сохранены в полном объеме.

---

## Предлагаемые изменения по компонентам

### 1. Редизайн главной страницы (`src/components/Homepage.js`)
- **Удаление**:
  - Убрать неинформативный большой приветственный баннер-заглушку.
  - Убрать громоздкий блок обратной связи (фидбека) и заменить его на компактный виджет.
- **Новая структура**:
  - **Quick Engineering Search**: Вывести центральное поле поиска с плейсхолдером: `"Search engineering reference data..."` (RU: `"Поиск по инженерным справочникам..."`).
  - **Quick Access Cards**: Сетка из 8 компактных, выровненных карточек быстрого доступа:
    1. Tubulars (НКТ и обсадные трубы)
    2. Thread Connections (Резьбовые соединения)
    3. Elastomers & Seals (Эластомеры и уплотнения)
    4. Steel Grades & Corrosion (Марки стали и коррозия)
    5. Standards Cross-Match (Сопоставление стандартов)
    6. Failure Encyclopedia (Энциклопедия отказов)
    7. Engineering Calculators (Инженерные калькуляторы)
    8. Running Recommendations (Руководство по спуску)
  - **Field Status Summary**: Компактный системный баннер, отображающий:
    - Автономность (Offline Ready)
    - Базу данных (Database Status)
    - Полевой режим (Field Mode Status)
    - Версию ПО и датасета (Version / Dataset Version)
  - **Small Feedback Area**: Минималистичный блок: `"Missing engineering reference data?"` с кнопкой `"Request Data"`, открывающей диалог ввода запроса.

### 2. Выравнивание боковой панели (`src/components/Sidebar.js`)
- Сделать высоту строк и внутренние отступы (padding) всех элементов меню строго одинаковыми.
- Обеспечить левое выравнивание иконок и текста, убрать любые смещения и "скачки" при смене вкладок.
- Привести к единой шрифтовой иерархии (typography hierarchy) для всех пунктов меню.

### 3. Удаление нумерации разделов (`src/components/EngineeringCard.js`)
- Полностью убрать порядковые номера (1, 2, 3...) из заголовков разделов карточек характеристик.
- Использовать строго профессиональные названия разделов:
  - *Overview* (Обзор / Описание)
  - *Technical Data* (Технические характеристики)
  - *Applications* (Области применения)
  - *Chemical Compatibility* (Химическая совместимость)
  - *Failure Modes* (Виды отказов)
  - *Standards* (Стандарты спецификаций)
  - *Engineering Notes* (Инженерные примечания)
  - *CAD / Engineering Diagrams* (Чертежи / Диаграммы CAD)

### 4. Доработка CAD-рендерера и заглушек (`src/components/DiagramRenderer.js` & `EngineeringCard.js`)
- Полностью исключить вывод сырых сообщений об ошибках ("Diagram missing" / "CAD missing").
- Если чертеж присутствует в базе: рендерить SVG чертеж через `DiagramRenderer`.
- Если чертеж отсутствует в базе:
  - Для физического оборудования (трубы, резьбы, эластомеры) выводить структурированную плашку:
    `"This engineering diagram is under validation."` (Чертеж находится на инженерной верификации).
    И отображать список: `"This section normally contains: dimensional drawing, cross-section, sealing profile, geometry, operational zones"`.
  - Для нефизических категорий (стали, отказы, калькуляторы) — полностью скрывать раздел чертежей.

### 5. Реконструкция химической совместимости (`src/components/EngineeringCard.js`)
- **Трубы (Tubulars)**: Заменить простой список на инженерную матрицу совместимости в виде таблицы:
  Столбцы: `Fluid` | `Compatibility` | `Risk` | `Notes`
  Строки: `H₂S`, `CO₂`, `Chlorides` (Хлориды), `Acidizing fluids` (Кислоты), `Brines` (Солевые растворы), `Completion fluids` (Растворы заканчивания).
- **Эластомеры и уплотнения (Elastomers & Seals)**: Вывести наглядную диаграмму зависимости температуры от химической стойкости для 10 материалов: NBR, HNBR, FKM/Viton, FFKM/Kalrez, AFLAS, EPDM, PTFE, PEEK, XNBR, PU. Добавить расшифровку легенды.
- **Стали (Steel Grades)**: Таблица коррозионной стойкости: `H₂S`, `CO₂`, `Chlorides`, Кислотная среда, Соответствие стандарту `NACE MR0175`, Риск растрескивания `SSC/HIC`.
- **Резьбы (Thread Connections)**: Полностью убрать раздел химической совместимости за неимением инженерного смысла.

### 6. Модернизация руководства по спуску (`src/modules/running-data/view.js`)
- Переименовать модуль в `"Completion Running Guide"` (RU: `"Руководство по спуску колонны"`).
- Наполнить детальной инженерной информацией:
  - Скорость спуска (Running speed limits)
  - Лимиты оборотов ротора (RPM limits)
  - Лимиты крутящего момента (Torque limits)
  - Натяг/разгрузка крюка (Pick-up / Slack-off margins)
  - Рекомендации по циркуляции (Circulation recommendations)
  - Операционные риски и лучшие практики (Operational risks & Best practices)
  - Блок рекомендаций "Делать / Не делать" (Do / Don't recommendations).

### 7. Калькуляторы (`src/modules/running-data/view.js`)
- Выровнять все метки (labels), поля ввода (inputs) и отступы по фиксированной сетке, исключить скачки интерфейса при обновлении результатов расчетов.
- Добавить краткие инженерные пояснения под каждым заголовком калькулятора (например, для Hydrostatic Pressure: *"Calculates hydrostatic pressure based on fluid density and measured depth."*).

### 8. Энциклопедия отказов (`src/components/EngineeringCard.js` / `src/modules/failures/view.js`)
- Добавить четкие **Триггерные условия (Triggering Conditions)**: сероводород, углекислота, хлориды, температура, превышение крутящего момента, усталость металла, ошибки спуска.
- Интегрировать визуальные схемы механизмов разрушений (SSC, HIC, промыв уплотнения, питтинговая коррозия, сход резьбы, усталостный излом, смятие, разрыв) вместо пустых мест.

### 9. Редизайн Диагностики (`src/modules/system-health/view.js`)
- Переименовать модуль из IT-термина "Diagnostics" в понятный инженеру `"System Status"` (RU: `"Операционный статус системы"`).
- Сгруппировать данные по разделам:
  - **Device Status**: Offline Ready, Database Healthy, Field Mode, Update Status.
  - **Engineering Library Health**: Общее число записей, статус целостности и дата последней проверки.
  - **Engineering Coverage**: Сводная статистика по разделам (трубы, резьбы, эластомеры, стали, отказы, стандарты).
  - **Missing Coverage**: Разделы и параметры, находящиеся в процессе инженерной верификации и наполнения базы.
- Исключить технический сленг разработчиков (программные логические ветки, adjacency list и др.).

### 10. Замена графа связей (`src/modules/system-health/view.js`)
- Удалить сложный для восприятия плоский граф связей.
- Заменить его на интерактивный **Проводник инженерных совместимостей (Engineering Relationship Explorer)**, позволяющий проследить цепочки совместимостей, например: Марка стали (L80) → Совместимые резьбы (VAM TOP) → Совместимые эластомеры (HNBR) → Условия Sour Service.

### 11. Замена прочерков интерактивными плейсхолдерами
- Исключить любые пустые прочерки `"—"` в карточках характеристик.
- Использовать интеллектуальные инженерные статусы:
  - *"Not applicable"* (Не применимо)
  - *"Under engineering validation"* (На верификации)
  - *"Data currently unavailable"* (Данные отсутствуют)
  - *"Requires engineering verification"* (Требуется проверка)

### 12. Видимость в светлой теме
- Провести аудит контрастности всех текстовых полей, бэджей и рамок в светлой теме, гарантируя полное соответствие WCAG AA (>4.5:1) при ярком солнечном свете.

## План верификации

### Автоматизированные тесты
- `npm run build` — сборка проекта для продакшена.
- `npx vite-node scratch/run_formula_tests.js` — сверка математических расчетов калькуляторов.
- `node scratch/hadalbore_db_completeness_auditor.js` — проверка полноты базы данных.
- `node scratch/hadalbore_svg_auditor.js` — верификация схем и чертежей.
- `node scratch/hadalbore_e2e_auditor.js` — headless E2E-тестирование контрастности и доступности UI.
